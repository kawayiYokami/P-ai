# 20260502_用户异步委托入口计划

## 背景

当前委托主要由 LLM 工具调用或用户 `@人格` 链路触发。用户希望能在当前会话中直接发起异步委托，并能从当前会话的消息多选菜单中把选中消息作为背景材料带给子代理。

## 目标

1. 当前会话内支持用户手动发起异步委托。
2. 消息多选菜单新增“发起委托”入口。
3. 首版只提供“审查”预设。
4. 前端记忆最近 3 次用户发起的异步委托配置，便于快速复用。
5. 选中消息作为背景材料传给子代理，只保留用户/助手可见纯文本，过滤工具调用、工具思维链和内部元数据。
6. 委托完成后结果固定写回当前会话。

## 非目标

1. 不做会话列表跨会话选择。
2. 不新增危险执行能力；子代理仍只返回内容。
3. 不改变 LLM 自己调用 `delegate` 工具的语义。

## 实现计划

1. 后端新增用户异步委托 command，接收当前 conversation_id、目标 department_id、预设/背景/问题/关注重点、选中 message ids。
2. 后端按 message ids 从当前会话抽取纯文本背景，过滤 tool 消息和 provider/tool 元数据。
3. 后端复用现有 delegate 记录创建和后台调度，`notify_assistant_when_done=false`，结果写回当前会话。
4. 前端消息多选栏增加“发起委托”按钮。
5. 增加轻量弹窗，支持选择目标部门、审查预设、编辑背景/问题/关注重点，并显示最近 3 次委托快捷项。
6. 更新 `CHANGELOG.md` 并做最小相关验证。
