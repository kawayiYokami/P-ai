# 20260502_delegate 工具简化计划

## 背景

当前 LLM 可见的 `delegate` 工具参数偏向“任务表单”，包含 `task_name`、`instruction`、`background`、`specific_goal`、`deliverable_requirement`、`notify_assistant_when_done` 等字段。实际使用中，LLM 发起的委托更像向某个部门的子代理咨询问题，子代理只返回内容，不应由模型控制危险执行或完成后续跑主助理。

## 目标

1. 将 LLM 可见 schema 简化为子代理咨询语义：
   - `department_id`
   - `mode`
   - `background`
   - `question`
   - `focus`
2. 字段保留简短说明，明确这是子代理咨询。
3. 不兼容旧字段；模型继续传 `instruction` 等旧字段时应失败。
4. 保留底层委托结果写回后可续跑主助理的运行时能力，但 LLM 自己发起的 async delegate 固定不续跑主助理。
5. 不迁移 delegate 存储表结构，降低归档历史与展示改动范围。

## 实现要点

1. 修改 `DelegateToolArgs` 为新字段。
2. 修改 `delegate` 工具 schema 与描述。
3. 校验逻辑将：
   - `question` 映射为内部 `instruction`
   - `background` 原样保存
   - `focus` 映射为内部 `specific_goal`
   - `deliverable_requirement` 保存为空
   - 标题从 `question` 自动截取
   - LLM async 委托 `notify_assistant_when_done=false`
4. 更新内部直接构造 `DelegateToolArgs` 的调用点。
5. 更新 `CHANGELOG.md`。
