# 05. 验收与测试计划

## 5.1 单元测试
- schema 初始化测试。
- MemoryRecord CRUD 测试。
- Tag/Rel 约束测试（唯一键、外键）。
- FTS upsert/delete/rebuild 测试。
- 遗忘算法阈值边界测试。

## 5.2 集成测试
- 端到端：写入记忆 -> 检索命中 -> 更新 -> 删除。
- 端到端：写入笔记标签 -> note 检索命中。
- 重建测试：删 FTS -> rebuild -> 命中恢复。
- 巡检测试：故意制造不一致 -> 自动修复。

## 5.3 回归测试
- 聊天主流程不因记忆重构退化。
- 归档总结与 memory-save 工具路径可用。
- UI memory viewer 列表/编辑/删除可用。

## 5.4 性能验收基线
- Memory 检索 P95 < 120ms（本地数据集基线）。
- Note 检索 P95 < 150ms。
- FTS 全量重建在目标样本规模下可接受（给出实测报告）。

## 5.5 质量门槛（发布前）
- 必须通过：
  - 功能测试 100%
  - 数据一致性巡检 100%
  - 备份恢复演练 100%
- 可延期：
  - 向量层相关能力（本期非目标）

## 5.6 发布后观察窗口
- 观察期 7 天。
- 每日检查：重建失败率、查询失败率、异常日志。
- 若出现系统性问题，回滚到上个版本应用包（非数据回滚）。
