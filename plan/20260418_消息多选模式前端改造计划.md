# 20260418_消息多选模式前端改造计划

## 一、文档目的

这份文档按当前已经实现的版本重写，用来替代最初“这一轮只做前端、不接后端”的旧计划。

当前这套多选能力已经不是纯前端壳子，而是已经打通：

- 消息多选
- 复制
- 派生
- 投送
- 忙态遮罩

同时也明确了这轮没有继续保留的内容：

- 删除
- 归档弹窗里的投放到会话
- 分享的真实落地链路

## 二、最终需求收口

这轮需求经过多次调整后，最终收口为：

1. 在聊天消息操作区提供多选入口
2. 进入多选模式后，让用户选择“具体哪些消息”
3. 输入框区域切换成多选操作条
4. 多选后的能力包括：
   - 派生新会话
   - 复制为格式化文本
   - 投送到其他会话
   - 分享（当前仅提示暂不支持）

也就是说，当前多选模式已经从“前端选中演示态”升级成“可执行动作态”。

## 三、当前已实现能力

### 1. 多选入口

多选入口位于：

- 单条消息底部操作区
- 在复制按钮左侧

当前使用单独图标作为“进入多选模式”入口。

### 2. 多选模式下的消息表现

进入多选模式后：

- 所有消息都进入可选状态
- 勾选控件显示在消息左侧
- 对用户消息和助理消息统一处理
- 点击整条消息行都可以切换勾选
- 选中高亮是整行高亮，不再只高亮消息气泡

### 3. 输入区替换

进入多选模式后，原输入区会切换成操作条。

当前操作条包含：

- `派生`
- `复制`
- `分享`
- `投送`
- `取消`

其中：

- 未选中任何消息时，动作按钮保持不可用
- 至少选中一条消息后，动作按钮才可点击

### 4. 复制

复制功能已经真实可用。

当前复制口径：

- 按选中顺序汇总消息
- 每条消息格式化为：

```text
[角色名]: 发言内容
```

补充规则：

- 图片显示为本地化的数量标签
- 语音显示为本地化的数量标签
- 附件显示为本地化的文件名列表
- 复制失败时会捕获剪贴板异常并给出失败状态

### 5. 分享

分享目前只保留占位行为：

- 点击后提示“分享功能暂不支持”

这条能力当前没有后端落地。

### 6. 派生

多选派生已经打通到后端。

当前派生行为是：

1. 从当前会话里提取已选消息
2. 新会话继承当前会话的主要设置：
   - 部门
   - 人格
   - 计划模式
   - Todo
   - 工作区
3. 新会话标题按以下规则生成：
   - 主会话：`主会话[派生自第N条对话]`
   - 普通会话：`原标题[派生自第N条对话]`
4. 新会话内容不是整会话克隆，而是：
   - 先复制源会话最新一条压缩消息
   - 再把已选原消息按原样复制进去
5. 原消息里的工具调用、`provider_meta`、`mcp_call` 会一起保留

这条派生链路当前不会额外调用 LLM 做摘要重写。

### 7. 投送

多选投送已经打通到后端。

当前投送行为是：

1. 点击 `投送`
2. 打开目标会话选择卡片
3. 选择目标会话后确认
4. 把已选原消息按原样插入目标会话末尾
5. 成功后自动切换到目标会话

当前投送会保留：

- 消息正文
- 工具调用
- `provider_meta`
- `mcp_call`

### 8. 忙态遮罩

以下动作执行时，聊天区会显示全屏遮罩和转圈：

- 派生
- 投送

这样可以避免用户误以为没有生效，也能在处理中冻结当前聊天交互。

## 四、当前约束与失败条件

### 1. 投送限制

如果目标会话处于以下状态，投送会失败：

- 助理正在流式输出
- 会话正在整理上下文

### 2. 目标会话选择

投送卡片打开时会先校验当前目标是否仍然有效：

- 若当前目标为空
- 或当前目标已不在可选会话列表里

则自动回退到第一个可用会话。

### 3. 分享限制

分享尚未接入任何真实分发链路，因此当前只允许展示提示，不会真正发送。

## 五、当前移除与不保留项

以下内容在讨论中出现过，但当前版本已经去掉：

- 多选删除按钮
- 删除后端命令
- 归档弹窗里的“投放到会话”卡片
- “只做前端演示，不接后端”这一旧范围

## 七、验收口径

当前版本应按下面这些点验收：

1. 点击消息底部多选入口后，可进入多选模式
2. 点击整条消息行可切换勾选
3. 输入区会替换成 `派生 / 复制 / 分享 / 投送 / 取消`
4. 复制会生成格式化文本并写入剪贴板
5. 分享会提示暂不支持
6. 派生会基于已选消息创建新会话，并保留最新压缩消息 + 原消息
7. 投送会把已选消息插到目标会话末尾，并在成功后切换过去
8. 派生和投送期间会显示全屏忙态遮罩

## 八、后续边界

这份计划当前不覆盖：

- 消息多选删除
- 分享真实落地
- 选区导出为文件
- 选区生成摘要卡
- 选区跨会话批量投送

如果后续继续扩展多选能力，应另起新计划，不再沿用“前端改造”这个旧范围描述。
