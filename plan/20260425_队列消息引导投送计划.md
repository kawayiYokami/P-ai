# 20260425_队列消息引导投送计划

## 目标

为现有聊天消息队列增加一种特殊状态：`引导中`。

它不是新消息来源，不是历史消息重放，也不是普通排队消息，而是：

- 现有队列消息被用户显式升级为 `引导中`
- 当前 LLM 轮次不被立刻打断
- 等当前轮次自然返回后，立即：
  - 收口当前轮次并正常落盘
  - 把所有 `引导中` 队列消息正式插入会话历史
  - 立刻重新激活下一轮 LLM

## 设计边界

### 1. 引导消息的来源

- 只允许来自现有 `pending_queue`
- 不新增新的输入来源
- 不从历史消息上直接打标

### 2. 引导消息的状态语义

- 队列消息新增状态：
  - `normal`
  - `guided`
- 用户点击“引导”后：
  - 队列项状态从 `normal` 变成 `guided`
  - 前端显示为 `引导中`
  - 一旦进入 `guided`，不能退回 `normal`

### 3. 当前轮次的处理方式

- 不像“停止”那样立刻打断当前轮
- 要等待当前 LLM 自然返回
- 当前轮结果仍按正常流程落盘

### 4. 引导接管的触发点

触发点固定在：

- 当前轮已经返回
- 当前轮状态即将从 `AssistantStreaming` 切回 `Idle`
- 释放该会话 claim 之前

此时如果发现该会话存在 `guided` 队列消息，则：

- 不走普通 `claim_queued_conversation_batches`
- 不等待下一轮普通队列调度
- 直接在当前会话内完成一次“引导批次”历史落地
- 紧接着立刻重新激活 LLM

### 5. 引导批次的处理规则

- 只消费当前会话中状态为 `guided` 的队列消息
- 保持原队列顺序
- 普通 `normal` 队列消息保持不动
- 引导批次写入历史后，立即从队列中移除

### 6. 前端表现

队列区：

- 普通用户队列消息新增“引导”按钮
- 点下后按钮变成 `引导中`
- `guided` 状态不可回退

流式区：

- 后端完成“当前轮落盘 + 引导消息插入”后，要继续沿用现有 `history_flushed`
- 前端收到后：
  - 当前 assistant 流式气泡转正
  - 引导消息被插入正式消息列表
  - 然后进入下一轮新的流式

### 7. 队列清理

- 只有在后台已经成功把 `guided` 消息写入历史并重新激活后，才从队列移除
- 若引导接管失败：
  - `guided` 消息保留在队列中
  - 不 silently 丢失

## 后端实现分段

### A. 数据结构

- 给 `ChatPendingEvent` 增加队列模式字段
- 给 `ChatQueueEventSummary` 增加前端显示所需状态字段
- 保持旧消息默认值为 `normal`

### B. 队列命令

- 新增“把某条队列消息升级为 guided”的命令
- 只允许：
  - 指定事件存在
  - 事件来源是 `user`
  - 当前状态不是 `guided`

### C. 调度接管

- 在当前轮完成后的会话收口处，检查当前会话是否存在 `guided` 队列消息
- 若存在：
  - 只抽取 guided 子集
  - 执行一次专用的 history flush
  - 立即重新激活下一轮
- 若不存在：
  - 保持现有普通队列处理逻辑

### D. 失败策略

- guided 提升失败：直接报错，队列状态不变
- guided 接管失败：记录日志，guided 消息保留
- 不做静默降级为 normal

## 前端实现分段

### A. 队列模型

- `ChatQueueEvent` 增加 `mode`
- 支持 `normal | guided`

### B. 队列 UI

- 用户消息显示：
  - 放回输入框
  - 删除
  - 引导 / 引导中
- 非用户消息不显示引导按钮

### C. 引导后的历史刷新

- 继续复用现有 `history_flushed`
- 不额外引入新的整页 reload 模型
- 只在必要时补一个轻量事件，用于告知“这批 guided 已正式插入”

## 日志要求

至少记录：

- 队列消息升级为 guided
- guided 接管开始
- guided 接管完成
- guided 接管失败

字段至少包含：

- conversation_id
- event_id / event_count
- 当前轮 request_id（如果有）
- 耗时
- 失败原因

## 最小验证

1. 当前会话空闲时，把队列消息设为 guided，应立即写入历史并启动下一轮。
2. 当前会话流式中时，把队列消息设为 guided，应等待本轮自然结束后再接管。
3. 多条 guided 消息应按原队列顺序插入。
4. guided 消息写入成功后，应从队列消失。
5. normal 队列消息不应被误消费。
6. guided 状态不能退回 normal。
