# 20260418_会话分叉复制与回归投放计划

## 一、文档目的

这份文档按当前已经落地的实现重写，用来替代最初那版“绑定会话 / 子会话 / 归档回归投放”的旧方案描述。

当前版本的核心已经收口为：

1. 新建会话时支持“继承当前会话”
2. 会话派生关系只认“父会话”
3. 归档统一产出“结论汇报”
4. 分叉会话归档时，只总结分叉点之后的讨论
5. “归档投放到会话”不再作为当前前台主流程

## 二、最终需求收口

本轮需求在多次讨论后，已经从最初的“会话复制 + 归档回归投放”收口成两条独立能力：

- 会话层：
  - 新建会话允许继承当前会话
  - 会话之间只记录父关系与分叉点
  - 归档的对象不再是过程快照，而是结论汇报
- 消息层：
  - 用户可从已选消息派生新会话
  - 用户可把已选消息直接投送到别的会话

也就是说：

- “回归会话”这套 UI 口径已经不再是当前主入口
- 当前前台主流程不再强调“归档时顺手投放”
- 归档负责沉淀结论
- 投送负责把选中的原始消息追加到目标会话

## 三、当前已实现范围

### 1. 新建会话支持“继承当前会话”

创建会话弹窗现在支持勾选：

- `继承当前会话`

启用后会：

- 复制当前前台会话的消息链
- 复制当前会话的人格、部门、计划模式、Todo、工作区等设置
- 新会话写入：
  - `parent_conversation_id = 来源会话.id`
  - `fork_message_cursor = 复制时最后一条消息`

这一条能力用于“整会话继承”，仍然保留。

### 2. 会话关系模型收口

本方案的前台语义已经收口为：

- `parent_conversation_id`
- `fork_message_cursor`

用户侧不再把“绑定会话 / 子会话”作为这套功能的主要概念。

当前前台行为是：

- 派生/继承会话只记录父会话
- 归档弹窗不再展示父子关系卡片
- 归档流程也不再要求用户在弹窗里选择“投放目标会话”

### 3. 归档语义统一为“结论汇报”

归档现在的统一语义是：

- 归档不是保存过程
- 归档主产物是一份结论汇报
- 目标是回答：`我们得出了什么结论？`

当前弹窗主动作已经收口为：

- 压缩
- 丢弃
- 归档

其中：

- `压缩`：整理较早历史，保留当前会话继续聊
- `丢弃`：直接删除当前会话，不生成归档
- `归档`：生成结论汇报并提炼记忆，保留为归档记录

### 4. 分叉会话的归档范围

如果会话带有有效的 `fork_message_cursor`，归档时会只总结分叉点之后的讨论。

当前后端口径：

- 普通会话：总结整个会话
- 分叉会话：只总结 `fork_message_cursor` 之后的消息

这样可以避免把父会话原有上下文又重复归档一遍。

### 5. 归档投放的当前状态

最初方案里的“归档并投放到目标会话”并没有作为当前前台最终主流程保留下来。

当前状态是：

- 归档仍然生成结论汇报
- 归档弹窗前台不再提供“投放到会话”卡片
- 前台主路径里的跨会话内容流转，已经转移到“消息多选投送”

换句话说：

- 当前产品语义里，归档和投送已经拆开
- 归档负责形成报告
- 投送负责把选中的原始消息插到目标会话末尾

## 四、后端实现口径

### 1. 会话字段

这套能力当前实际依赖：

- `parent_conversation_id`
- `fork_message_cursor`

其中：

- `parent_conversation_id`
  - 表示当前会话从哪条会话派生而来
- `fork_message_cursor`
  - 表示分叉点
  - 用于归档时裁剪“只总结分叉后的讨论”

### 2. 继承当前会话

后端在创建继承会话时会：

- 克隆消息
- 重置新会话主键与时间字段
- 保留前台工作所需设置
- 写入父会话与分叉点

### 3. 归档范围裁剪

归档链路当前会先判断：

- `fork_message_cursor` 是否存在
- 该 cursor 是否仍能在当前消息链里找到

若能找到：

- 只取 cursor 之后的消息参与归档汇报

若找不到：

- 回退为总结整个会话

## 五、前端实现口径

### 1. 新建会话弹窗

当前 UI 已包含：

- 会话标题输入
- 部门选择
- `继承当前会话` 勾选项

### 2. 处理当前会话弹窗

当前 UI 已收口为：

- 压缩卡片
- 丢弃卡片
- 归档卡片
- 底部显示消息数与上下文占用

当前不再显示：

- 父会话卡片
- 投放到会话卡片

## 六、与旧方案的差异

以下口径已经不再作为当前实现说明：

- 绑定会话
- 子会话列表
- 归档时手动选择投放目标会话
- 把归档汇报作为当前前台主路径回灌到别的会话

这些设计曾在讨论中出现过，但已经被后续实现收口替换。

## 七、验收口径

当前这份计划对应的验收点应理解为：

1. 新建会话时可以继承当前会话
2. 继承/派生会话会记录父会话与分叉点
3. 归档统一生成结论汇报
4. 分叉会话归档时只总结分叉点之后的讨论
5. 处理当前会话弹窗不再展示归档投放目标

## 八、后续边界

这份计划当前不承诺以下内容：

- 会话树形 UI
- 父子链可视化
- 归档完成后自动把报告再投回父会话
- 多目标回灌
- 归档报告编辑器

如果后续要恢复“归档回投”路线，应另起新计划，不再沿用这版旧口径。
